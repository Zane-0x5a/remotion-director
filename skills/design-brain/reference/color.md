# 色彩设计

> 你亲手写每个 OKLCH/hex——下面是落色时该记住的策略与知觉。用 **OKLCH** 工作(L 感知明度 / C 饱和 / H 色相),它比 hex 好推理明度与对比。

## 先选策略,再选色(commitment 四档)[原文]

**Pick a color strategy before picking colors.**
- **Restrained**:tinted neutrals + 一个 accent ≤10%。产品默认;品牌极简。
- **Committed**:一个饱和色承载 30–60% 画面。身份驱动。
- **Full palette**:3–4 个具名角色,各有意使用。品牌战役;数据可视。
- **Drenched**:画面**就是**那个色。品牌 hero;高情绪拍。

15s 竖屏(深色场居多):Restrained / Committed 是常态;Drenched 适合强情绪 hero 拍(整屏被一个色浸透)。策略可逐拍 override 全局(一支 Restrained 的片里某个高潮拍 Drenched),但要是有意的。

## Theme via scene sentence(逼出 dark/light)[原文]

Dark vs. light **never a default**. 不是"工具深色显酷",不是"浅色保险"。选之前**写一句物理场景:谁用、在哪、什么环境光、什么情绪。如果这句话逼不出答案,就是不够具体,加细节直到能逼出。**

## AI 默认配色警告(必背)[原文]

- **The cream / sand / beige body bg is the saturated AI default of 2026.** 整条暖中性带(OKLCH L 0.84-0.97, C<0.06, hue 40-100)无论叫 `paper/cream/sand/linen/parchment/ivory` 都读作"纸"。brief 说"warm/traditional/magazine-warm/editorial" 时,**别把它翻译成近白暖染底色——那就是 AI 动作**。"温暖"靠 accent + 字体 + 材质扛,不靠 body bg。
- **Tinted neutrals**:朝品牌色相加 0.005–0.015 chroma。别无脑朝暖/冷默认染(跨项目单一文化的动作)。

## 色彩在相邻/运动中的知觉(Albers 同时对比 · 必读)

一支视频里**相邻关系逐帧在变**,色彩不能当孤立色板评,要在**上下文**里评:

- **同时对比**:同一个色,旁边的色会改变它的观感——① 中明度块在亮场上显得更暗、在暗场上显得更亮;② 红场会让相邻中性灰**泛绿**(互补诱导);③ 暖橙放在更暖的红上会显得**偏冷**(色彩减法)。**一个 accent 的真实观感取决于它的邻居,不是它的孤立色卡。**
- **Bezold 效应**:小面积色**向大面积包围色同化**,大面积色则对比。点缀色必须在实际包围环境里判断。
- **边界振动**:等明度的互补色直接相邻 → 边缘闪烁。可用于 1-3 秒高能瞬间,**持续=眼疲劳**。别让需要静读的文字长时间压在等明度互补场上。
- **时间性色彩对比**:拍 N 收暖、拍 N+1 开冷 → 对比被**时间相邻放大**。可主动利用(叙事转折处故意做温度切换,如 dusk→night),也可能突兀——你编色彩旅程时主动决定它是哪种。
- **Alpha 是设计气味**:大量用透明度去"凑"中间调 = 色板不完整。**显式定义中间调,别靠半透明叠出来。**

## 对比(可读性硬地板)

被读的核心文案对背景 ≥4.5:1(大字号 ≥3:1)。**最常见失败 = 浅灰文字压浅色底——"为了优雅用浅灰"是 AI 设计读不清的头号原因。** 接近就把文字推向墨色端。氛围/whisper 文字(低对比)可以是有意的设计,但别把**该被读的核心文案**做成低对比。

## 色彩旅程按拍

每拍一色彩状态 + 拍间溶解,不是全片一个种子色。从叙事 + 情绪一次性生成整条旅程(色彩可以是叙事的:从冷夜升到暖烬,让场在变)。
