# 构图设计

> 你亲手决定每个块的位置/大小/对齐/留白——下面是落笔时该记住的视知觉机制(C 层有视知觉实证,非审美品味)。让"你想要的解读"成为观众脑中**最省力的那个解读**,而不是和知觉对着干。

## 落笔记住(知觉机制)

- **单焦点(仅当有 hero)**:同屏只允许一个最高强调层。**两个都想当 hero → 拆开,或降级其一**。并列/平权/网格(无 hero)不强制单焦点——强行单焦点是品味入侵。
- **分组 = 邻近**:语义相关的块挨近、不相关的拉开(组内距 < 组间距,约 1.5-2×)。
- **视觉重心别偏**:多块时别全堆屏幕一侧/上 1/3(典型负样本就是三块堆上 1/3、质心偏高)。要么质心居中、要么沿画面高度展开。
- **阅读路径(locale-aware)**:F/Z 流向是 LTR/RTL 镜像的工程默认,非普世真理;别和 locale 反向。
- **视觉权重 ≠ 阅读位置**:视觉权重大(无论来自字阶、尺寸、对比还是强调)不等于"该放上面"。"标题在上、CTA 在下"是产品语义,要显式安排,别让次要块因权重低冒到顶部。
- **留白是有意的**:有害死白(大片无意义空带、质心被甩偏)vs 故意留白(居中 framing 的呼吸)**几何同形、靠意图区分**。你要留白就让它明显是设计决定(对称/居中/沿轴展开),别留成"没排满"的事故。

## 格式塔知觉机制(视知觉实证,非品味)

- **分组原则按知觉强度排序**:connectedness(连线/同容器)> common fate(同向运动)> proximity(邻近)> common region(同背景区)> 颜色相似 > 尺寸/形状/连续/闭合。**两原则冲突时强者赢**:若靠邻近分了组、却让其中一块独自运动(common fate),运动会把它从邻近组里**拽出来**。别让弱原则的分组被强原则拆散,除非你就想拆。
- **图形-背景(figure-ground)**:被读成"图形"(前景)的块通常是——面积更小、更被包围、更凸、更对称、对比更高。**非故意的图底翻转令人迷失**:当某块在动画中变大/变亮到反超,它可能从前景塌成背景(或反之)。scale/contrast 随时间变时,留意别触发意外翻转。
- **视觉层级完整排序**:motion > size > contrast > saturation > position > isolation。**想让 A 比 B 更重,优先动 A、或放大 A,而非只靠把 A 放更高的位置**(position 在序列里靠后,单靠位置抢不过尺寸/对比)。
- **静态↔动态分组开关**:静止时邻近(proximity)主导分组;**一旦运动,common fate 接管**。入场/退场的运动会临时重组观众看到的"组",别让它和你的语义分组打架。
- **Prägnanz(最简解读)**:大脑选最省力的解释。一块在左消失、相似块在右出现 → 被读成"同一块移动了"(而非两块)。**要观众读成"移动"还是"换了一块",就让那个解读成为最简的**。
- **组内:组间 < 1:2 = 边界含糊**:组内间距相对组间距太大,分组就读不出来。保证语义分组"组内明显比组间紧"。

## 对齐 / 光学

- **共线最大化、错位最小化**:能落同一条线的边就落同一条线;容器/关键对齐一致。
- **数学包围盒 ≠ 视觉包围盒**:大字号下标点(引号/逗号)和升降部(g/y/p)会撑大数学边界,只按数学 bbox 算对齐/质心 → 人眼判"偏"。按**字干**(非 bounding box)对齐才不显偏。
- **别默认居中一切**:`大字居中` 是最强的品类反射。左对齐做编辑感、偏轴做张力都是合法选择——center 只是其中一个,不是默认。

## 跨轴耦合(和运动/节奏共管)

- **运动是最大视觉权重**:静态算出的焦点,若另一块用剧烈入场,它运动的 0.5-1.5s 内视觉权重瞬间反超 → 打破单焦点。**最高静态权重元素须配最高动画能量,或严格错峰**。
- **同位轮换 vs 排死逐亮 vs 流动顶走**:任意时刻屏上几个在场、位置是静态还是流动,直接决定构图——是摆 1 个、摆 N 个静态全景、还是算几阶段版面 + 位移。开篇就想清楚。

## 安全区

关键文本别贴帧边/出血(竖屏给四边留足边距,核心文案在内 ~80%)。别把要被读的核心文案设计成"顶到边"。

> 真品味墙(留白量 30% 还是 40% / 对称 vs 偏轴 / 网格松紧 / type scale 取 1.2 还是 1.5)——A∧C 可行域内的残余,无客观对错,按你的整体风格定调,但别冒充"客观最优"。
